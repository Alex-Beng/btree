A simple example

The library provides associative containers that are resident on some form of direct access external memory, such as a hard-disk, solid-state disk, or USB memory-stick.  Here is an example, set1.cpp:

#include <boost/btree/btree_set.hpp>
#include <iostream>
using std::cout;
using namespace boost::btree;

int main()
{
  btree_set<int> set("set.btr", flags::truncate);

  set.insert(5);
  set.insert(3);
  set.insert(1);

  for (btree_set<int>::iterator it = set.begin();
    it != set.end(); ++it)
    cout << *it << '\n';

  cout << "lower_bound(3) is " << *set.lower_bound(3) << '\n';
  cout << "upper_bound(3) is " << *set.upper_bound(3) << '\n';

  return 0;
}

This should be familiar to users of the C++ standard library associative container std::set. The only real difference, other than names, is the constructor call, high-lighted in yellow. Because the container is disk-resident, an argument supplies a path for the file to be created. A second argument supplies flags that specify details for opening the file.

Build and test set1.cpp like this (Windows):

>cd boost-root\libs\btree\example
>b2 set1 install
... build output elided ...
>bin\set1
1
3
5
lower_bound(3) is 3
upper_bound(3) is 5

Go ahead and try this yourself! Once you have built and tested set1.cpp as is, try experimenting with some of the other functions already familiar to you from std::set. Did they work as you expected? Hopefully you answer was "yes"!

But what about the additional template parameters, constructor arguments, and member functions you may have spotted if you looked at the referenced documentation? Don't worry about them for now; the defaults will get you started, and many of the unfamiliar parameters, arguments, and functions are useful mostly to advanced users tuning for ultimate performance.

What you probably do want to learn about right away is how to cope with the requirements on Key (and T, when we look at maps).
